Scenario details

Background: Non Repudiation Service or NRS is a solution that provides the ability for HMRC to accept data in a way
where the submitter cannot argue in a legal setting that "That wasn't what I sent you!". NRS as an end to end solution must stand up to scrutiny in a court of law.

In the context of NRS the API platform needs to be able to demonstrate that it does not alter the payload that it receives in any way. This includes encoding, whitespace, etc...

NRS requires that each request that sends data to a service that uses NRS must sent a SHA2-256 hash of the body in a separate header.

We must make sure that when a service receives a message it can generate the same hash value as the client did. This should prove the body hasn't changed and is the way NRS works.

For more information on NRS see https://confluence.tools.tax.service.gov.uk/display/NR/How+to+use+Non-Repudiation+services
